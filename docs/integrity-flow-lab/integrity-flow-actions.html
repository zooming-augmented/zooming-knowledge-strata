Integrity Flow Actions — 初級編

v5 / 2026-05-04 — ← Integrity Flow Lab

input（起点）からoutput（発信）まで、AIと一緒に小さく動くための行動フロー。
目標：記事の60〜80%は自分の言葉・体験・感情から書く。AIはつなぎと引き出し役。

Day 1 環境・起点を整える ／ Day 2 素材・設計・骨格を作る ／ Day 3 書いて、出す

Day 1

STEP 0｜起点の確認

  • Direction：向かいたい方向を仮置きする。曖昧でOK。
  • テーマ：今日書きたいこと・試したいことを1つ決める。
  • 読者：届けたい相手のイメージを確認する。
  • 決まっているものはそのまま入力。「まだ」のものだけAIと一緒に決める。

Day 2

STEP 1｜素材を集める ＋ 場面を引き出す

  • 素材確認：体験・気づき・参考情報のうち、手元にあるものを選ぶ。
  • 場面を引き出す（重要）：「その素材の中で、感情が一番動いた瞬間・場面はいつか？」を1つ言葉にする。ここが記事の核になる。
  • まだ何もない場合：STEP 4で今日の小さな試行を決めてから書く。

STEP 2｜設計する（誰に・何を・どこで）＋ マーケティング

2-A｜マーケティングリサーチ（任意）

  • AIに調べてもらう / 自分の調査結果を貼る / スキップ　の3択。
  • note・X・Webの「競合」と「空白地帯」を確認する。

2-B｜記事を設計する

  • 切り口：AIに2〜3案出させて選ぶ。
  • 独自性の確認（重要）：「この切り口、あなたにしか言えない理由は何か？」を1つ言葉にする。答えが出ない場合は切り口を変える。
  • 成功の定義：「この記事を読んだ人が次の5分でやりたくなることは何か？」を1文で言えるか確認する。
  • 書く範囲：1段落 / 構成と一言メモ / 500字 / 全部　の中から今日の範囲を決める。

STEP 3｜動けない理由を1つ見つける

  • 時間 / 何を書くか迷っている / 公開への不安 / 完成への自信 のどれかを1つ選ぶ。
  • AIに「このボトルネックを崩す最小の一歩は何か」を聞く。

STEP 4｜今日のアクションを1つ決める

  • 5〜15分でできる形に小さくする。できなければさらに分割する。
  • 「今日はここまで」を明示する。完璧な計画より、動ける計画を優先する。

Day 3

STEP 5｜実践 ＋ 自分の言葉チェック

  • 計画した最初の一歩を実行する。70点で次のステップへ進む。
  • 自分の言葉チェック（重要）：書いた内容のうち、自分の経験・感情から書いた部分はどこか確認する。60%未満なら1か所だけ自分の言葉に書き直す。

STEP 6｜振り返り

  • 3行記録：今日やったこと / 気づき / 次にやること
  • 人間比率の確認：今日書いたもので、自分にしか書けない部分はどこか記録する。次回はその割合を少し増やすことを意識する。
  • Direction更新：実践を通じて変わったことを反映する。

STEP 7｜Output（任意）

  • 公開できる部分だけ外に出す。全部出す必要はない。
  • 発信の形：note / X / GitHub Pages / 社内メモ / 誰かへのシェア
  • 発信した内容を次の起点（Direction・テーマ）にする。

原則

  • 上から順にやる必要はない。詰まったステップから始めてよい。
  • 成功の定義：次の一歩が決まり、次の循環に入れたら成功。
  • このリストは使うたびに育てる。tune our flow — 小さな一歩を、集めよう
